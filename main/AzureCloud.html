AZ-204 Cheat Sheet

Azure Cloud and Spring Boot

  • What is Azure Cloud?
    Azure Cloud is a cloud computing service created by Microsoft for building, testing, deploying, and managing applications and services through Microsoft-managed data centers. It provides a wide range of cloud services, including computing, analytics, storage, and networking.
  • How do you deploy a Spring Boot application to Azure?
    You can deploy a Spring Boot application to Azure using Azure App Service, Azure Spring Cloud, or Azure Kubernetes Service (AKS). For example, using Azure App Service, you can create a new web app, configure the deployment settings, and deploy your Spring Boot application using Git, FTP, or a CI/CD pipeline.
  • What is Azure Spring Cloud?
    Azure Spring Cloud is a fully managed service for Spring Boot applications. It simplifies the deployment, management, and scaling of Spring Boot applications on Azure. It provides built-in support for service discovery, configuration management, and monitoring.
  • How do you create an Azure Spring Cloud service?
    You can create an Azure Spring Cloud service using the Azure portal, Azure CLI, or Azure Resource Manager templates. For example, using the Azure portal, you can navigate to the Azure Spring Cloud service, click "Create", and follow the steps to configure the service and deploy your Spring Boot application.
  • What is Azure App Service?
    Azure App Service is a fully managed platform for building, deploying, and scaling web apps. It supports multiple programming languages, including Java, and provides features such as auto-scaling, load balancing, and integrated CI/CD.
  • How do you configure a Spring Boot application for Azure App Service?
    To configure a Spring Boot application for Azure App Service, you need to create a web app in the Azure portal, configure the deployment settings, and deploy your application using Git, FTP, or a CI/CD pipeline. You can also use the Azure Maven Plugin to deploy your application directly from your development environment.
  • What is Azure Kubernetes Service (AKS)?
    Azure Kubernetes Service (AKS) is a managed Kubernetes service that simplifies the deployment, management, and scaling of containerized applications. It provides features such as automated upgrades, monitoring, and scaling.
  • How do you deploy a Spring Boot application to Azure Kubernetes Service (AKS)?
    To deploy a Spring Boot application to Azure Kubernetes Service (AKS), you need to containerize your application using Docker, create a Kubernetes deployment and service configuration, and deploy the application to your AKS cluster using kubectl or a CI/CD pipeline.
  • What is the Azure Maven Plugin?
    The Azure Maven Plugin is a tool that allows you to deploy Java applications to Azure from your development environment. It supports deploying to Azure App Service, Azure Spring Cloud, and Azure Kubernetes Service (AKS).
  • How do you use the Azure Maven Plugin to deploy a Spring Boot application?
    To use the Azure Maven Plugin to deploy a Spring Boot application, you need to add the plugin to your pom.xml file, configure the deployment settings, and run the mvn azure-webapp:deploy command to deploy your application to Azure App Service.
  • What is Azure Key Vault?
    Azure Key Vault is a cloud service for securely storing and accessing secrets, such as API keys, passwords, certificates, and cryptographic keys. It helps safeguard cryptographic keys and secrets used by cloud applications and services.
  • How do you integrate Azure Key Vault with a Spring Boot application?
    You can integrate Azure Key Vault with a Spring Boot application by using the Azure Key Vault Spring Boot Starter. Add the dependency to your pom.xml file, configure the Key Vault properties, and use the @Value annotation to inject secrets into your application.
  • What is Azure Blob Storage?
    Azure Blob Storage is a service for storing large amounts of unstructured data, such as text or binary data. It is designed for storing and serving large amounts of data, such as images, videos, and backups.
  • How do you upload a file to Azure Blob Storage using a Spring Boot application?
    To upload a file to Azure Blob Storage using a Spring Boot application, you need to add the Azure Storage Blob dependency to your pom.xml file, configure the storage account properties, and use the BlobServiceClient to upload the file.
  • What is an Azure Storage Account?
    An Azure Storage Account is a container that provides a unique namespace for storing data objects in Azure Blob Storage, Azure File Storage, Azure Queue Storage, and Azure Table Storage. It allows you to manage and access your storage resources.
  • How do you create an Azure Storage Account?
    You can create an Azure Storage Account using the Azure portal, Azure CLI, or Azure Resource Manager templates. For example, using the Azure portal, you can navigate to the Storage Accounts service, click "Create", and follow the steps to configure and create the storage account.
  • What is Azure Service Bus?
    Azure Service Bus is a fully managed enterprise message broker that allows you to decouple applications and services. It provides reliable message queuing and publish-subscribe messaging capabilities.
  • How do you send and receive messages using Azure Service Bus in a Spring Boot application?
    To send and receive messages using Azure Service Bus in a Spring Boot application, you need to add the Azure Service Bus dependency to your pom.xml file, configure the Service Bus properties, and use the ServiceBusSenderClient and ServiceBusReceiverClient to send and receive messages.
  • What is Azure Event Hub?
    Azure Event Hub is a fully managed, real-time data ingestion service that can stream millions of events per second. It is designed for big data and analytics scenarios, such as telemetry data from IoT devices and application logs.
  • How do you send and receive events using Azure Event Hub in a Spring Boot application?
    To send and receive events using Azure Event Hub in a Spring Boot application, you need to add the Azure Event Hubs dependency to your pom.xml file, configure the Event Hub properties, and use the EventHubProducerClient and EventHubConsumerClient to send and receive events.
  • What is Azure SQL Database?
    Azure SQL Database is a fully managed relational database service built on Microsoft SQL Server. It provides high availability, scalability, and security for your database workloads.
  • How do you connect a Spring Boot application to Azure SQL Database?
    To connect a Spring Boot application to Azure SQL Database, you need to add the JDBC driver dependency to your pom.xml file, configure the database connection properties in the application.properties file, and use Spring Data JPA or JDBC to interact with the database.
  • What is Azure Cosmos DB?
    Azure Cosmos DB is a globally distributed, multi-model database service that provides high availability, low latency, and scalability. It supports multiple data models, including document, key-value, graph, and column-family.
  • How do you connect a Spring Boot application to Azure Cosmos DB?
    To connect a Spring Boot application to Azure Cosmos DB, you need to add the Azure Cosmos DB Spring Boot Starter dependency to your pom.xml file, configure the Cosmos DB properties, and use Spring Data Cosmos to interact with the database.
  • What is Azure Functions?
    Azure Functions is a serverless compute service that allows you to run event-driven code without provisioning or managing servers. It supports multiple programming languages, including Java, and can be triggered by various events, such as HTTP requests, timers, and messages.
  • How do you create an Azure Function in a Spring Boot application?
    To create an Azure Function in a Spring Boot application, you need to add the Azure Functions dependency to your pom.xml file, create a function class with the @FunctionName annotation, and deploy the function to Azure using the Azure Functions Maven Plugin.
  • What is Azure Logic Apps?

  • How do you integrate Azure Logic Apps with a Spring Boot application?

  • What is Azure DevOps?

  • How do you set up a CI/CD pipeline for a Spring Boot application using Azure DevOps?

  • What is Azure Monitor?
    Azure Monitor is a service for collecting, analyzing, and acting on telemetry data from your cloud and on-premises environments. It provides insights into the performance and health of your applications and infrastructure.
  • How do you monitor a Spring Boot application using Azure Monitor?

  • What is Azure Active Directory (AAD)?

Docker - Java Microservices

  • What is Docker?
    Docker is an open-source platform that automates the deployment, scaling, and management of applications using containerization. Containers are lightweight, portable, and run consistently across different environments.
  • What is a Docker container?
    A Docker container is a lightweight, standalone, and executable package that includes everything needed to run a piece of software, including the code, runtime, libraries, and dependencies.
  • How do you create a Dockerfile for a Java microservice?

    A Dockerfile is a text file that contains instructions for building a Docker image. Here is an example Dockerfile for a Java microservice:

    
    # Use an official OpenJDK runtime as a parent image
    FROM openjdk:11-jre-slim

    # Set the working directory in the container
    WORKDIR /app

    # Copy the JAR file into the container
    COPY target/my-microservice.jar /app/my-microservice.jar

    # Expose the port the application runs on
    EXPOSE 8080

    # Run the JAR file
    ENTRYPOINT ["java", "-jar", "my-microservice.jar"]
  • How do you build a Docker image for a Java microservice?

    To build a Docker image for a Java microservice, navigate to the directory containing the Dockerfile and run the following command:

    docker build -t my-microservice:latest .
  • How do you run a Docker container for a Java microservice?

    To run a Docker container for a Java microservice, use the following command:

    docker run -d -p 8080:8080 my-microservice:latest

    This command runs the container in detached mode and maps port 8080 of the container to port 8080 on the host machine.

  • How do you use Docker Compose to manage multiple Java microservices?

    Docker Compose is a tool for defining and running multi-container Docker applications. Here is an example docker-compose.yml file for managing multiple Java microservices:

    
    version: '3'
    services:
      service1:
    image: my-microservice1:latest
    ports:
      - "8081:8080"
      service2:
    image: my-microservice2:latest
    ports:
      - "8082:8080"

    To start the services, run the following command:

    docker-compose up -d
  • How do you scale Java microservices using Docker Compose?

    To scale Java microservices using Docker Compose, use the --scale option with the docker-compose up command. For example, to scale service1 to 3 instances, run:

    docker-compose up -d --scale service1=3
  • How do you monitor Docker containers running Java microservices?

    You can monitor Docker containers running Java microservices using tools like Docker stats, Prometheus, Grafana, and cAdvisor. For example, to view real-time statistics of running containers, use:

    docker stats

Redis and Spring Boot

  • What is Redis?
    Redis is an open-source, in-memory data structure store that can be used as a database, cache, and message broker. It supports various data structures such as strings, hashes, lists, sets, and more.
  • How do you integrate Redis with a Spring Boot application?
    To integrate Redis with a Spring Boot application, you need to add the Spring Data Redis dependency to your pom.xml file, configure the Redis connection properties, and use the RedisTemplate or StringRedisTemplate to interact with Redis.
  • What is the purpose of the @EnableRedisRepositories annotation?
    The @EnableRedisRepositories annotation is used to enable Redis repositories in a Spring Boot application. It allows you to create repository interfaces for Redis data access.
  • How do you use the @Cacheable annotation with Redis?
    The @Cacheable annotation is used to cache the result of a method in Redis. When the method is called, the result is stored in the cache, and subsequent calls with the same parameters return the cached result. For example: 
    @Service
public class MyService {
    @Cacheable(value = "myCache", key = "#id")
    public MyObject getMyObjectById(String id) {
        // Method implementation
    }
}
  • How do you use the @CachePut annotation with Redis?
    The @CachePut annotation is used to update the cache with the result of a method. It ensures that the cache is updated with the latest data. For example: 
    @Service
public class MyService {
    @CachePut(value = "myCache", key = "#id")
    public MyObject updateMyObject(String id, MyObject myObject) {
        // Method implementation
    }
}
  • How do you use the @CacheEvict annotation with Redis?
    The @CacheEvict annotation is used to remove an entry from the cache. It ensures that the cache is cleared when the annotated method is called. For example: 
    @Service
public class MyService {
    @CacheEvict(value = "myCache", key = "#id")
    public void deleteMyObject(String id) {
        // Method implementation
    }
}
  • How do you configure Redis in a Spring Boot application?
    To configure Redis in a Spring Boot application, add the following properties to your application.properties file: 
    spring.redis.host=localhost
spring.redis.port=6379
  • Example of using RedisTemplate in a Spring Boot application: 
    @Service
public class RedisService {
    private final RedisTemplate redisTemplate;

    @Autowired
    public RedisService(RedisTemplate redisTemplate) {
        this.redisTemplate = redisTemplate;
    }

    public void save(String key, Object value) {
        redisTemplate.opsForValue().set(key, value);
    }

    public Object get(String key) {
        return redisTemplate.opsForValue().get(key);
    }

    public void delete(String key) {
        redisTemplate.delete(key);
    }
}

Kubernetes - AKS

  • What is Kubernetes?
    Kubernetes is an open-source container orchestration platform that automates the deployment, scaling, and management of containerized applications. It provides a framework to run distributed systems resiliently.
  • What is Azure Kubernetes Service (AKS)?
    Azure Kubernetes Service (AKS) is a managed Kubernetes service provided by Microsoft Azure. It simplifies the deployment, management, and operations of Kubernetes by offloading much of the operational overhead to Azure.
  • How do you create an AKS cluster?
    You can create an AKS cluster using the Azure portal, Azure CLI, or Azure Resource Manager templates. For example, using the Azure CLI, you can create a cluster with the following command: 
    az aks create --resource-group myResourceGroup --name myAKSCluster --node-count 3 --enable-addons monitoring --generate-ssh-keys
  • How do you deploy a Java microservice to AKS?
    To deploy a Java microservice to AKS, you need to containerize your application using Docker, push the Docker image to a container registry, create Kubernetes deployment and service YAML files, and apply them to your AKS cluster using kubectl. For example: 
    kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
  • How do you scale a deployment in AKS?
    You can scale a deployment in AKS using the kubectl scale command. For example, to scale a deployment named my-deployment to 5 replicas, run: 
    kubectl scale deployment my-deployment --replicas=5
  • How do you monitor an AKS cluster?
    You can monitor an AKS cluster using Azure Monitor for containers, which provides insights into the performance and health of your cluster. You can also use tools like Prometheus and Grafana for more advanced monitoring and visualization.
  • How do you update a deployment in AKS?
    To update a deployment in AKS, you can modify the deployment YAML file and apply the changes using kubectl apply. For example: 
    kubectl apply -f updated-deployment.yaml
  • How do you roll back a deployment in AKS?
    You can roll back a deployment in AKS using the kubectl rollout undo command. For example, to roll back the last deployment of my-deployment, run: 
    kubectl rollout undo deployment my-deployment
  • How do you expose a deployment in AKS?
    You can expose a deployment in AKS by creating a Kubernetes service. For example, to expose a deployment named my-deployment as a LoadBalancer service, create a service YAML file and apply it using kubectl: 
    kubectl apply -f service.yaml
  • How do you manage secrets in AKS?
    You can manage secrets in AKS using Kubernetes Secrets. Create a secret using kubectl create secret and reference it in your deployment YAML file. For example: 
    kubectl create secret generic my-secret --from-literal=username=myuser --from-literal=password=mypassword
